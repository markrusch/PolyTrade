## 3. SYSTEM ARCHITECTURE (FACTUAL)

### 3.1 CLOB Model

* The Polymarket CLOB is a **hybrid-decentralized** exchange
* Orders are **off-chain signed messages** (EIP-712), not on-chain transactions
* Matching happens off-chain by the operator
* Settlement occurs **on-chain** via the CTF Exchange contract
* Orders are signed limit orders; market orders are limit orders with marketable prices

**Source:** https://docs.polymarket.com/developers/CLOB/introduction

---

### 3.3 Authentication Layers

#### Level 0: No Authentication (Public Data)
* Market data, orderbooks, prices
* No credentials required

#### Level 1 (L1): Private Key Authentication
* Required to create or derive API keys
* Uses wallet private key to sign EIP-712 messages
* Methods: `createApiKey()`, `deriveApiKey()`, `createOrDeriveApiKey()`

#### Level 2 (L2): API Key Authentication
* Required for order placement and management
* Uses credentials from L1: `{ apiKey, secret, passphrase }`
* All trading operations require L2 authentication

**Source:** https://docs.polymarket.com/developers/CLOB/authentication

---

### 3.4 Signature Types (Exact)

Orders must specify signature type based on wallet configuration:

| Value | Type | Description | Use Case |
|-------|------|-------------|----------|
| `0` | EOA | Externally Owned Account | MetaMask, hardware wallets, direct private key control |
| `1` | Polymarket Proxy | Email/Magic wallet proxy | Email login, Magic Link authentication |
| `2` | Gnosis Safe Proxy | Gnosis Safe smart contract | Safe multisig wallets, browser wallet proxies |

**Critical Implementation Detail:**
* For `signature_type=1` or `signature_type=2`, you **must** provide a `funder` parameter
* The `funder` is the proxy wallet address that holds funds
* The signing key is separate from the funder address
* EOA wallets (`signature_type=0`) do not require a `funder` parameter

**Source:** https://docs.polymarket.com/developers/CLOB/quickstart, https://github.com/Polymarket/py-clob-client

---

### 3.5 Order Types

| Order Type | Code | Description | Use Case |
|------------|------|-------------|----------|
| GTC | `OrderType.GTC` | Good-Til-Cancelled | Standard limit order, active until filled or cancelled |
| GTD | `OrderType.GTD` | Good-Til-Date | Limit order active until specified timestamp |
| FOK | `OrderType.FOK` | Fill-Or-Kill | Market order, must execute immediately in full or cancel |
| FAK | `OrderType.FAK` | Fill-And-Kill | Market order, execute immediately for available size, cancel remainder |

**Market Orders:**
* There is no dedicated "market order" type
* Use `FOK` or `FAK` with a marketable price
* For BUY orders: set price at or above current ask
* For SELL orders: set price at or below current bid

**Source:** https://docs.polymarket.com/developers/CLOB/orders/create-order

## 4. PREREQUISITES & TOKEN ALLOWANCES

### 4.1 Token Allowance Requirements

**Critical:** Before placing any orders, wallets must set token allowances:

#### For Buying (USDC Spending)
```typescript
// Approve USDC spending by CTF Exchange
await usdcContract.approve(
  EXCHANGE_ADDRESS,
  ethers.constants.MaxUint256  // Unlimited approval
);
```

#### For Selling (Conditional Token Transfer)
```typescript
// Approve CTF Exchange to transfer conditional tokens
await ctfContract.setApprovalForAll(
  EXCHANGE_ADDRESS,
  true  // Enable operator
);
```

#### Negative Risk Markets
For markets with `negRisk: true`, also approve:
```typescript
await ctfContract.setApprovalForAll(
  NEG_RISK_ADAPTER,
  true
);

await ctfContract.setApprovalForAll(
  NEG_RISK_EXCHANGE,
  true
);
```

**Note:** This is a **one-time setup per wallet**. Once set, allowances persist unless revoked.

**Source:** https://docs.polymarket.com/developers/CLOB/quickstart, https://docs.polymarket.com/developers/builders/relayer-client

---

### 4.2 Order Balance Constraints

Orders are subject to real-time balance monitoring:

```
maxOrderSize = underlyingAssetBalance - Σ(orderSize - orderFillAmount)
```

**Implications:**
* Cannot place orders exceeding available balance
* Open orders reserve funds until filled or cancelled
* System tracks balances, allowances, and on-chain cancellations in real-time
* Attempting to abuse these checks results in blacklisting

**Source:** https://docs.polymarket.com/developers/CLOB/orders/orders

## 6. GAMMA API (MARKET DISCOVERY)

### 6.1 Overview

Gamma is Polymarket's market metadata service. It indexes on-chain data and provides:
* Market discovery and search
* Event categorization
* Volume and liquidity metrics
* Market resolution data

**Base URL:** `https://gamma-api.polymarket.com`

**Source:** https://docs.polymarket.com/developers/gamma-markets-api/overview

```typescript
interface Market {
  id: string;
  question: string;
  slug: string;
  description: string;
```

### 7.2 Create Market Order

For immediate execution (FOK or FAK):


## 8. WEBSOCKET REAL-TIME DATA

### 8.1 CLOB WebSocket Overview

Polymarket provides two WebSocket channels:
* **Market Channel:** Order book updates, price changes (no auth required)
* **User Channel:** Personal order and trade updates (requires auth)

**Source:** https://docs.polymarket.com/developers/CLOB/websocket/wss-overview

---

### 8.2 Market Channel (Public)

**Endpoint:** `wss://ws-subscriptions-clob.polymarket.com/ws/market`

```typescript
import WebSocket from 'ws';

const ws = new WebSocket('wss://ws-subscriptions-clob.polymarket.com/ws/market');

ws.on('open', () => {
  const subscription = {
    markets: [tokenId],  // Array of token IDs
    type: "market"
  };
  ws.send(JSON.stringify(subscription));
});

ws.on('message', (data) => {
  const event = JSON.parse(data);
  
  switch(event.event_type) {
    case "book":
      // Order book update
      console.log("Book update:", event);
      break;
    case "price_change":
      // Price changed
      console.log("Price:", event.price);
      break;
    case "last_trade_price":
      // Last trade price
      console.log("Last trade:", event.price);
      break;
    case "tick_size_change":
      // Tick size changed
      console.log("Tick size:", event.tick_size);
      break;
  }
});
```

**Event Types:**
* `book`: Order book snapshot or update
* `price_change`: Market price changed
* `last_trade_price`: Last executed trade price
* `tick_size_change`: Tick size was updated

**Source:** https://docs.polymarket.com/developers/CLOB/websocket/market-channel

---

### 8.3 User Channel (Authenticated)

**Endpoint:** `wss://ws-subscriptions-clob.polymarket.com/ws/user`

```typescript
import WebSocket from 'ws';

const ws = new WebSocket('wss://ws-subscriptions-clob.polymarket.com/ws/user');

ws.on('open', () => {
  const subscription = {
    markets: [tokenId],
    type: "user",
    auth: {
      apiKey: apiCreds.apiKey,
      secret: apiCreds.secret,
      passphrase: apiCreds.passphrase
    }
  };
  ws.send(JSON.stringify(subscription));
});

ws.on('message', (data) => {
  const event = JSON.parse(data);
  
  switch(event.event_type) {
    case "order":
      // Order status update
      console.log("Order update:", event);
      break;
    case "trade":
      // Trade executed
      console.log("Trade:", event);
      break;
  }
});
```

**Event Types:**
* `order`: Order created, updated, filled, or cancelled
* `trade`: Trade was executed

**Source:** https://docs.polymarket.com/developers/CLOB/websocket/user-channel

### 8.4 Real-Time Data Streaming (RTDS)

**Alternative WebSocket for broader market data:**

**Endpoint:** `wss://ws-live-data.polymarket.com`

```typescript
import { RealTimeDataClient } from "@polymarket/real-time-data-client";

const onMessage = (message) => {
  console.log(`[${message.topic}:${message.type}]`, message.payload);
};

const onConnect = (client) => {
  console.log("Connected to RTDS");
  
  // Subscribe to trades
  client.subscribe({
    subscriptions: [{
      topic: "activity",
      type: "trades",
      filters: `{"market_slug":"will-bitcoin-hit-100k"}`
    }]
  });
  
  // Subscribe to crypto prices
  client.subscribe({
    subscriptions: [{
      topic: "crypto_prices",
      type: "update"
    }]
  });
};

const rtdsClient = new RealTimeDataClient({
  onMessage,
  onConnect
});

rtdsClient.connect();
```

**RTDS Topics:**
* `activity` (types: `trades`, `new_markets`)
* `crypto_prices` (type: `update`)
* `comments` (type: `*` for all)

**Source:** https://github.com/Polymarket/real-time-data-client

## 9. BUILDER RELAYER (GASLESS TRANSACTIONS)

### 9.1 Overview

The Builder Relayer enables:
* Gasless trading (no MATIC needed)
* Proxy wallet deployment
* Builder order attribution
* Token approvals without gas

**Base URL:** `https://relayer-v2.polymarket.com/`

**Source:** https://docs.polymarket.com/developers/builders/relayer-client

---

### 9.2 Builder Credentials

Builder API credentials consist of:
* `POLY_BUILDER_API_KEY`
* `POLY_BUILDER_SECRET`
* `POLY_BUILDER_PASSPHRASE`

Obtain these from: https://polymarket.com/settings?tab=builder

**Critical Security Note:**
* **Never expose builder credentials in client-side code**
* Use server-side signing or remote signing patterns
* Store credentials in environment variables or secrets manager

**Source:** https://docs.polymarket.com/developers/builders/builder-profile

---

```typescript

import { privateKeyToAccount } from "viem/accounts";
import { polygon } from "viem/chains";

const builderCreds: BuilderApiKeyCreds = {
  key: process.env.POLY_BUILDER_API_KEY!,
  secret: process.env.POLY_BUILDER_SECRET!,
  passphrase: process.env.POLY_BUILDER_PASSPHRASE!
};

const builderConfig = new BuilderConfig({
  localBuilderCreds: builderCreds
});

const account = privateKeyToAccount(process.env.PRIVATE_KEY as Hex);
const wallet = createWalletClient({
  account,
  chain: polygon,
  transport: http(process.env.RPC_URL)
});

// Initialize with SAFE transaction type (default)
const relayClient = new RelayClient(
  "https://relayer-v2.polymarket.com/",
  137,
  wallet,
  builderConfig,
  RelayerTxType.SAFE  // or RelayerTxType.PROXY
);
```

**Source:** https://github.com/Polymarket/builder-relayer-client

---

### 9.4 Relayer Transaction Types

| Type | Value | Description | Deployment |
|------|-------|-------------|------------|
| SAFE | `RelayerTxType.SAFE` | Gnosis Safe proxy wallet | Must call `deploy()` explicitly before first use |
| PROXY | `RelayerTxType.PROXY` | Polymarket custom proxy | Auto-deploys on first transaction |

**Choosing Transaction Type:**
* **SAFE:** For multisig wallets, enhanced security, explicit deployment control
* **PROXY:** For simpler use cases, automatic deployment, faster onboarding

**Source:** https://github.com/Polymarket/builder-relayer-client

---

### 9.5 Deploy Proxy Wallet

**For SAFE wallets only (PROXY auto-deploys):**

```typescript
const response = await relayClient.deploy();
const result = await response.wait();

if (result) {
  console.log("Safe deployed successfully!");
  console.log("Transaction Hash:", result.transactionHash);
  console.log("Safe Address:", result.proxyAddress);
}
```

**Check if Already Deployed:**
```typescript
const safeAddress = deriveSafe(eoaAddress, config.SafeContracts.SafeFactory);
const deployed = await relayClient.getDeployed(safeAddress);

if (!deployed) {
  await relayClient.deploy();
}
```

**Source:** https://docs.polymarket.com/developers/builders/relayer-client
